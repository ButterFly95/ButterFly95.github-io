Web 应用黑客手册/第三章：Web 应用技术

Web 应用技术

HTTP 协议

HTTP 请求

HTTP 请求报文一般包括以下几部分：

  1. HTTP 方法，一般都是 GET
  2. URL
  3. HTTP 版本
  4. Referer header，用于标识 URL 来源
  5. User-Agent header，客户端软件
  6. Host header，域名
  7. Cookie header，服务器发送给客户端的其他参数
HTTP 响应

HTTP 响应报文一般包括以下几部分：

  1. HTTP 版本
  2. 状态码
  3. 状态描述
  4. Server header，包含 Web 服务器信息
  5. Set-Cookie，用于设置 Cookie
  6. Pragma header，不要缓存
  7. Content-Type，文档类型
  8. Content-Length，文档长度
HTTP Methods
  1. GET，获取资源
  2. POST，执行操作
  3. HEAD，等同没有响应消息没有 message body 的 GET
  4. TRACE，用于诊断
  5. OPTIONS，获取支持的方法
  6. PUT，上传文件

作业

第一题

OPTIONS 方法有什么用？

获取服务端支持的请求方法

第二题

If-Modified-Since 和 If-None-Match 有什么用？为什么攻击时对此感兴趣？

If-Modified-Since：只有当请求的资源自从某个时间之后有改变才会返回该资源

If-None-Match：只有服务器没有匹配的 etag，服务端才会返回请求的资源

缓存中毒

第三题

服务端设置 cookie secure 标识有什么意义？

cookie 只能在 HTTPS 进行传输

第四题

301 和 302 有什么区别？

301 永久移除

302 短暂移除

第五题

使用 SSL 的时候，浏览器怎么和 HTTP proxy 交互？

HTTP CONNECT，proxy 仅作为 TCP 连接的中间节点
