Web 应用黑客手册/第十二章：攻击用户：跨站脚本

攻击用户：跨站脚本

XSS 分类

XSS 分为：反射型，存储型和 DOM 型。

反射型 XSS

直接把用户的输入插入 HTML 文档

  1. 用户登录
  2. 攻击者发送伪造的 URL 给用户
  3. 用户点击伪造的 URL
  4. 服务端返回带有攻击者 JavaScript 代码的响应报文
  5. 攻击者的脚本在用户浏览器执行
  6. 攻击者的脚本发送会话令牌给攻击者
  7. 攻击者得到会话令牌
存储型 XSS
  1. 攻击者发送恶意信息到服务端
  2. 用户登录
  3. 用户查看攻击者的信息
  4. 服务端返回带有攻击者 JavaScript 代码的响应报文
  5. 攻击者的脚本在用户浏览器执行
  6. 攻击者的脚本发送会话令牌给攻击者
  7. 攻击者得到会话令牌

攻击者需要诱导用户才能利用反射型 XSS 漏洞，但是，对于存储型 XSS，不需要这么麻烦。

DOM 型 XSS
  1. 用户登录
  2. 攻击者发送伪造的 URL 给用户
  3. 用户点击伪造的 URL
  4. 服务端返回包含 JavaScript 硬编码的脚本
  5. 浏览器处理返回的 HTML 文档，出发攻击者的脚本
  6. 攻击者的脚本发送会话令牌给攻击者
  7. 攻击者得到会话令牌
